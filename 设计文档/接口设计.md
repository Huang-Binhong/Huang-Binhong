- 路径使用复数名词，如 `/persons`、`/works`。
- 使用 HTTP 方法区分操作：
  - `GET` — 读取资源
  - `POST` — 新建资源
  - `PUT`/`PATCH` — 更新资源
  - `DELETE` — 删除资源
- 接口应为无状态（每次请求都包含所需信息）以保证可扩展性。
- 支持分页、过滤、排序，以便查询大量数据而不导致性能问题。
- 错误响应应使用标准 HTTP 状态码，并提供结构化错误信息。
- API 版本管理（如 `/api/v1/…`）以保证未来可变更。

---

#### 通用响应格式（示例）

```
{
  "code": 0,
  "message": "success",
  "data": { … }
}
```

- `code`：整型，0 表示成功，非 0 表示错误
- `message`：文本，成功或错误信息
- `data`：对象、数组或 null，具体接口返回数据

#### 错误码规范（示例）

| HTTP 状态码 | code | 描述             |
| ----------- | ---- | ---------------- |
| 400         | 4001 | 请求参数校验失败 |
| 401         | 4011 | 认证失败／未登录 |
| 403         | 4031 | 无权限访问       |
| 404         | 4041 | 资源不存在       |
| 500         | 5001 | 内部服务器错误   |

#### 接口清单（主要资源）

以下为各资源接口示例，包含路径、方法、参数、说明、响应：

**1. 人物 (Person)**

- `GET /api/v1/persons`
  - 描述：获取人物列表
  - 查询参数（可选）：`page`（int，默认 1）、`pageSize`（int，默认 20）、`search`（字符串，按姓名或别号模糊）
  - 返回：分页对象，`items` 为 Person 对象数组
- `POST /api/v1/persons`
  - 描述：新增人物（管理员）
  - 请求体：Person 对象（JSON）
  - 返回：新建的 Person 对象
  - 认证：需 Bearer Token
- `GET /api/v1/persons/{personId}`
  - 描述：获取某个人物详情
  - 路径参数：`personId`（字符串）
  - 返回：Person 对象
- `PUT /api/v1/persons/{personId}`
  - 描述：更新人物信息（管理员）
  - 请求体：Person 对象
  - 返回：更新后的 Person 对象
  - 认证：需 Bearer Token
- `DELETE /api/v1/persons/{personId}`
  - 描述：删除人物（管理员）
  - 返回：无数据（或 204 状态）
  - 认证：需 Bearer Token

**2. 生平事件 (Event)**

- `GET /api/v1/persons/{personId}/events`
  - 描述：获取某人物的生平事件列表
  - 路径参数：`personId`
  - 返回：事件数组，按 year 升序
- `POST /api/v1/persons/{personId}/events`
  - 描述：新增事件（管理员）
  - 请求体：Event 对象
  - 认证：需 Bearer Token
- `PUT /api/v1/events/{eventId}`
  - 更新事件
- `DELETE /api/v1/events/{eventId}`
  - 删除事件

**3. 地点 (Location)**

- `GET /api/v1/locations`
  - 获取所有地点列表
- `GET /api/v1/locations/{locationId}`
  - 获取某地点详情
- `POST /api/v1/locations`
  - 新增地点（管理员）
- `PUT /api/v1/locations/{locationId}`
- `DELETE /api/v1/locations/{locationId}`

**4. 人际关系 (Relation)**

- `GET /api/v1/relations`
  - 获取所有关系
  - 支持查询参数：`type`（关系类型）、`fromPersonId`、`toPersonId`
- `GET /api/v1/persons/{personId}/relations`
  - 获取指定人物的所有关系
- `POST /api/v1/relations`
  - 新增关系（管理员）
- `PUT /api/v1/relations/{relationId}`
- `DELETE /api/v1/relations/{relationId}`

**5. 作品 (Work)**

- `GET /api/v1/works`
  - 获取作品列表
  - 查询参数：`page`、`pageSize`、`category`、`stylePeriod`、`material`、`search`
  - 返回：分页对象，`items` 为 Work 对象
- `GET /api/v1/works/{workId}`
  - 获取作品详情
- `POST /api/v1/works`
  - 新增作品（管理员）
- `PUT /api/v1/works/{workId}`
- `DELETE /api/v1/works/{workId}`

#### 常用查询参数说明

- `page`: 整数，当前页码，默认 1
- `pageSize`: 整数，每页条数，默认 20
- `search`: 字符串，模糊搜索关键词（如姓名、作品标题）
- `category`, `stylePeriod`, `material`: 字符串，作为筛选条件
- `sortBy`: 字符串，指定排序字段，如 `creationDate_desc` 或 `title_asc`
